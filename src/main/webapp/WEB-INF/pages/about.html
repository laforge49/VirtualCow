About Virtual COW

Virtual COW is a high-performance, robust, scalable, versioning, transactional graphical database. Nodes in the graph are mapped onto immutable AA trees which support versioning and may be larger than available RAM.

Virtual COW takes its name from the virtualized AA trees, parts of which are rolled into RAM as needed. And from the use of a Copy-On-Write (COW) technique to implement the database.

Virtual COW is scalable in four ways:

  1. Vertical scalability is achieved through the use of JActor2 in place of traditional threads/locks. Virtual COW will make full use of all the processor cores it has access to.
  2. Many simultaneous users are supported through the use of Jetty and Servlet API 3.1. Open connections do not tie up threads. And queries are processed in parallel using a common cache.
  3. The database will scale up to a million blocks of a million bytes each (a terabyte).
  4. Other than the size of the database, there are no constraints on the number of nodes in the graph nor on the number of links that connect to a node. There is a practical limit on the number of links that originate with a node, but this is a limit in the design of the displays, not a limit of the underlying database.

Robustness is achieved through the use of JActor2, which monitors message processing, prevents race conditions and implicitly orders data locks.

Virtual COW is based on two open source projects, JActor2 and utils.
